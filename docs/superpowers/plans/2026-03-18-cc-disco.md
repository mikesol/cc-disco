# cc-disco Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a minimal Discord-to-Claude-Code router where each Discord thread maps to a Claude Code session.

**Architecture:** Single TypeScript file (`server.ts`) in `.claude/skills/server/`. Discord.js listens for messages, spawns Claude Code with `--resume` and a deterministic UUID v5 session ID derived from the thread ID, streams output back via Discord message edits. Cron via `croner` reading `cron.json`. Three skill docs (server, cron, op).

**Tech Stack:** TypeScript, discord.js, uuid (v5), croner, child_process

**Spec:** `docs/superpowers/specs/2026-03-18-cc-disco-design.md`

const processes = new Map<string, ChildProcess>();

function spawnClaude(sessionId: string, message: string): ChildProcess {
  const args = [
    '-p',
    '--resume', sessionId,
    '--dangerously-skip-permissions',
    '--output-format', 'stream-json',
    '--verbose',
    '--model', CLAUDE_MODEL,
    message,
  ];
  return spawn(CLAUDE_BIN, args, {
    cwd: process.cwd(),
    stdio: ['ignore', 'pipe', 'pipe'],
  });
}

- [ ] **Step 2: Add stream-json parser**

```typescript
function parseStreamEvents(proc: ChildProcess, onDelta: (delta: string) => void, onDone: (result: string) => void): void {
  let buffer = '';
  proc.stderr!.on('data', () => {}); // drain stderr to prevent pipe blocking
  proc.stdout!.on('data', (chunk: Buffer) => {
    buffer += chunk.toString();
    const lines = buffer.split('\n');
    buffer = lines.pop() ?? '';
    for (const line of lines) {
      if (!line.trim()) continue;
      try {
        const evt = JSON.parse(line);
        if (evt.type === 'content_block_delta' && evt.delta?.type === 'text_delta') {
          onDelta(evt.delta.text);
        } else if (evt.type === 'result' && evt.result) {
          onDone(evt.result);
        }
      } catch { /* skip unparseable lines */ }
    }
  });
  proc.on('exit', () => onDone(''));
}
```

```typescript
async function handleMessage(threadId: string, content: string): Promise<void> {
  const sessionId = threadSessionId(threadId);
  const channel = await client.channels.fetch(threadId);
  if (!channel?.isTextBased() || !('send' in channel)) return;

  // Interrupt if busy
  const existing = processes.get(threadId);
  if (existing) {
    existing.kill('SIGINT');
    await new Promise<void>((resolve) => {
      const timeout = setTimeout(() => { existing.kill('SIGKILL'); resolve(); }, 5000);
      existing.on('exit', () => { clearTimeout(timeout); resolve(); });
    });
    processes.delete(threadId);
  }

  // Send placeholder
  const reply = await channel.send('...');

  // Spawn Claude
  const proc = spawnClaude(sessionId, content);
  processes.set(threadId, proc);

  let accumulated = '';
  let lastEdit = 0;
  const EDIT_INTERVAL = 1500;

  const editReply = async (final: boolean) => {
    const now = Date.now();
    if (!final && now - lastEdit < EDIT_INTERVAL) return;
    lastEdit = now;
    const text = accumulated.slice(0, 2000) || '...';
    try { await reply.edit(text); } catch { /* ignore edit failures */ }
  };

  parseStreamEvents(
    proc,
    (delta) => { accumulated += delta; void editReply(false); },
    async (result) => {
      processes.delete(threadId);
      const final = result || accumulated || '(no output)';
      // Final edit — split if needed
      if (final.length <= 2000) {
        try { await reply.edit(final); } catch {}
      } else {
        try { await reply.edit(final.slice(0, 2000)); } catch {}
        for (let i = 2000; i < final.length; i += 2000) {
          try { await channel.send(final.slice(i, i + 2000)); } catch {}
        }
      }
    },
  );
}
```

import { Cron } from 'croner';
import { readFileSync } from 'node:fs';

type CronJob = { schedule: string; threadId: string; message: string };
let activeCrons: Cron[] = [];

function loadCrons(): void {
  for (const c of activeCrons) c.stop();
  activeCrons = [];

  let jobs: CronJob[];
  try {
    jobs = JSON.parse(readFileSync('cron.json', 'utf-8'));
  } catch {
    console.log('No cron.json found or invalid — skipping cron setup');
    return;
  }
  for (const job of jobs) {
    const cron = new Cron(job.schedule, () => {
      const prefix = processes.has(job.threadId)
        ? '[Scheduled task — if you\'re mid-task, finish your current work first, then handle this]\n\n'
        : '';
      void handleMessage(job.threadId, prefix + job.message);
    });
    activeCrons.push(cron);
    console.log(`Cron registered: "${job.message.slice(0, 50)}" → ${job.schedule}`);
  }
}
```

## Build

## Run

## Restart (if installed as systemd service)
systemctl --user restart cc-disco

## Status
systemctl --user status cc-disco

## Logs
journalctl --user -u cc-disco -f

## How it works
- Listens to Discord messages from allowed users
- Messages in channels auto-create threads
- Each thread maps to a Claude Code session (UUID v5 from thread ID)
- Spawns `claude -p --resume <session-id> --dangerously-skip-permissions`
- Streams response back via Discord message edits (~1.5s interval)
- New message while busy → SIGINT current process → resume with new message

# Cron

Scheduled tasks are defined in `cron.json` at the repo root. The server reads it on startup and re-reads on SIGHUP.

## Format

```json
[
  { "schedule": "0 7 * * *", "threadId": "123456789", "message": "Morning vibe check" },
  { "schedule": "0 17 * * *", "threadId": "987654321", "message": "Evening digest" }
]
```

- `schedule` — standard 5-field cron expression (parsed by croner)
- `threadId` — Discord thread ID where the message will be sent
- `message` — the prompt sent to Claude Code in that thread

## Adding a job
1. Edit `cron.json` — add an entry
2. Send SIGHUP to reload: `kill -HUP $(pgrep -f 'node dist/server.js')`
   Or restart: `systemctl --user restart cc-disco`

## Busy threads
If the target thread has a running Claude Code process when the cron fires, the message is prefixed with: `[Scheduled task — if you're mid-task, finish your current work first, then handle this]`

## Getting a thread ID
Enable Developer Mode in Discord (User Settings → Advanced), then right-click a thread → Copy Thread ID.
```
